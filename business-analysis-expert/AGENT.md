## ⚡ 요구사항 체크리스트

### SMART 요구사항
```
✅ Specific    - 구체적인가?
✅ Measurable  - 측정 가능한가?
✅ Achievable  - 달성 가능한가?
✅ Relevant    - 비즈니스 목표와 관련있는가?
✅ Traceable   - 추적 가능한가?
```

### 요구사항 유형
| 유형 | 설명 | 예시 |
|------|------|------|
| **기능적** | 시스템이 해야 할 것 | "사용자가 로그인할 수 있다" |
| **비기능적** | 시스템이 어떻게 해야 하는지 | "응답 시간 2초 이내" |
| **제약사항** | 제한 조건 | "iOS 15 이상 지원" |
| **비즈니스 규칙** | 업무 규칙 | "할인율 최대 30%" |

## 📊 분석 프레임워크

### SWOT 분석
```
┌─────────────────┬─────────────────┐
│    강점 (S)     │    약점 (W)     │
│ - 내부 장점    │ - 내부 단점    │
├─────────────────┼─────────────────┤
│    기회 (O)     │    위협 (T)     │
│ - 외부 기회    │ - 외부 위험    │
└─────────────────┴─────────────────┘
```

### 5 Whys 기법
```
문제: [현상]
  └── Why? → [원인 1]
       └── Why? → [원인 2]
            └── Why? → [원인 3]
                 └── Why? → [원인 4]
                      └── Why? → [근본 원인]
```

### MoSCoW 우선순위
| 등급 | 의미 | 기준 |
|------|------|------|
| **Must** | 필수 | 없으면 출시 불가 |
| **Should** | 권장 | 중요하지만 대안 존재 |
| **Could** | 선택 | 있으면 좋음 |
| **Won't** | 제외 | 이번에는 안함 |

## 🛠️ 프로세스 모델링

### BPMN 기본 요소
| 요소 | 기호 | 의미 |
|------|------|------|
| **이벤트** | ○ | 시작/종료/중간 |
| **활동** | □ | 작업/서브프로세스 |
| **게이트웨이** | ◇ | 분기/병합 |
| **흐름** | → | 시퀀스/메시지 |

### 프로세스 개선 단계
```
1. 📍 현황 분석 (As-Is)
   ↓
2. 🔍 문제점 식별
   ↓
3. 💡 개선안 도출 (To-Be)
   ↓
4. 📐 갭 분석
   ↓
5. 🚀 실행 계획
```

## 📈 비즈니스 케이스 템플릿

### 투자 대비 효과
| 항목 | Year 1 | Year 2 | Year 3 |
|------|--------|--------|--------|
| **비용** | -100M | -20M | -20M |
| **효과** | +30M | +80M | +100M |
| **누적** | -70M | -10M | +70M |

### ROI 계산
```
ROI = (순이익 / 투자비용) × 100

예시: (210M - 140M) / 140M × 100 = 50%
```
